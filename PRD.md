# 健康关怀系统 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 项目简介

| 项目 | 说明 |
|------|------|
| **产品名称** | 健康关怀系统 (Health Care System) |
| **产品定位** | 面向抑郁患者的综合心理健康管理移动应用 |
| **目标用户** | 抑郁症患者（核心用户）、心理医生/咨询师（专业用户）、患者家属（潜在用户） |
| **核心价值** | 通过整合标准化心理量表评估、智能手环健康数据同步、情绪追踪等功能，为用户提供全方位的健康关怀服务 |

### 1.2 痛点需求

| 痛点 | 现状 | 解决方案 | 价值 |
|------|------|---------|------|
| **心理评估不规范** | 缺乏专业可靠的自我评估工具 | 提供标准化量表（SDS/SAS/PSS/PHQ-9） | 专业、可追溯的评估结果 |
| **健康数据分散** | 生理数据和心理数据无法有效整合 | 整合手环数据与心理数据 | 全面了解身心健康状况 |
| **情绪波动难追踪** | 难以实时记录情绪变化 | EMA 生态瞬时评估 | 及时捕捉情绪波动 |
| **数据记录不连续** | 患者难以坚持长期记录健康数据 | 智能提醒 + 简化填报流程 | 提高数据记录依从性 |
| **医患沟通不畅** | 医生难以实时了解患者状态 | 医生端数据查看功能 | 促进医患协同管理 |

## 3. 技术架构

### 3.1 前后端分离架构

| 层级 | 技术栈 | 说明 | 职责 |
|------|--------|------|------|
| **前端表现层** | uni-app + Vue.js 2.x | 跨平台应用开发框架 | 页面渲染、用户交互、数据展示 |
| **前端UI层** | uView UI | uni-app UI组件库 | 提供统一的UI组件 |
| **前端可视化层** | ECharts | 数据可视化图表库 | 图表渲染、数据可视化 |
| **前端状态管理** | Vuex | 状态管理 | 全局状态管理、数据共享 |
| **后端接口层** | Spring Boot 2.3.4 | Java后端框架 | RESTful API接口、业务逻辑 |
| **后端安全层** | Shiro + JWT | 安全认证框架 | 身份认证、权限控制 |
| **后端业务层** | Spring Service | 业务服务层 | 业务逻辑处理、数据处理 |
| **后端持久层** | MyBatis 2.1.3 | ORM框架 | 数据库操作、SQL映射 |
| **数据库层** | MySQL 8.0+ | 关系型数据库 | 数据存储、数据查询 |
| **连接池** | Druid 1.1.8 | 数据库连接池 | 连接管理、性能优化 |
| **API文档** | Swagger/Knife4j 2.9.2 | API文档工具 | 接口文档生成、在线调试 |
| **消息推送** | 极光推送 3.7.6 | 推送服务 | 消息推送、通知提醒 |
| **数据集成** | Python 3.10+ | 脚本语言 | Garmin数据同步 |
| **第三方API** | Garmin Connect API | 智能手环API | 健康数据获取 |

### 3.3 数据流架构

| 数据流向 | 说明 | 技术实现 |
|---------|------|---------|
| **用户操作 → 前端** | 用户交互触发事件 | Vue.js事件处理 |
| **前端 → 后端** | HTTP请求发送数据 | uni.request + RESTful API |
| **后端 → 数据库** | 数据持久化存储 | MyBatis + MySQL |
| **数据库 → 后端** | 数据查询返回 | MyBatis查询映射 |
| **后端 → 前端** | JSON数据响应 | Spring Boot响应封装 |

| **Garmin手环 → 云端** | 手环数据同步 | Garmin Connect App |
| **云端 → Python脚本** | 定时数据拉取 | Python + Garmin API |
| **Python脚本 → 后端** | 数据提交 | HTTP POST请求 |
| **后端 → 推送平台** | 消息推送 | 极光推送SDK |
| **推送平台 → 用户设备** | 通知送达 | 厂商推送通道 |

| **医生查看患者流程** | 1. 医生登录 → 2. 查看患者列表 → 3. 选择患者 → 4. 查看患者详情 → 5. 查看手环数据 → 6. 查看量表结果 → 7. 查看情绪记录 → 8. 查看日常报告 → 9. 生成综合评估 | 医生管理患者数据 | 患者列表页、患者详情页 |
| **邀请码生成流程** | 1. 医生登录 → 2. 进入邀请码管理 → 3. 点击生成邀请码 → 4. 设置有效期（默认30天） → 5. 生成6位数字码 → 6. 复制邀请码 → 7. 分享给患者 | 医生邀请患者注册 | 邀请码管理页 |
| **Garmin数据同步流程** | 1. 用户佩戴手环 → 2. 手环采集数据 → 3. 蓝牙同步到App → 4. 上传到Garmin云端 → 5. Python脚本定时拉取（每小时） → 6. 数据处理格式化 → 7. 提交到后端API → 8. 存储到MySQL → 9. 前端查询展示 | 自动同步手环数据 | 后台自动执行 |

### 4.2 异常流程

| 异常场景 | 处理流程 | 说明 |
|---------|---------|------|
| **登录失败** | 1. 提示错误信息 → 2. 密码错误5次锁定30分钟 → 3. 提供忘记密码入口 | 保护账号安全 |
| **网络异常** | 1. 检测网络状态 → 2. 提示网络错误 → 3. 提供重试按钮 → 4. 使用本地缓存数据 | 提升用户体验 |
| **数据同步失败** | 1. 记录失败日志 → 2. 下次重试 → 3. 超过3次失败提示用户 | 确保数据完整性 |
| **推送失败** | 1. 记录失败原因 → 2. 切换推送通道重试 → 3. 记录推送日志 | 提高推送到达率 |
| **Token过期** | 1. 检测Token有效期 → 2. 自动刷新Token → 3. 刷新失败跳转登录页 | 保持登录状态 |
